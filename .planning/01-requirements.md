# 01-requirements.md — 요구사항 정의 (MoSCoW)

**버전**: 1.0 | **날짜**: 2026-05-12

---

## MoSCoW 우선순위 설명

| 레벨 | 의미 |
|------|------|
| **Must Have** | 반드시 구현. 없으면 앱이 의미 없음 |
| **Should Have** | 강력히 권장. 없어도 동작하지만 경험이 크게 저하됨 |
| **Could Have** | 있으면 좋음. 시간 여유 시 구현 |
| **Won't Have** | 현재 범위에서 제외. 미래 버전 고려 |

---

## Must Have (핵심 기능)

### M-01. 할 일(퀘스트) 관리
- [ ] 퀘스트 등록: 제목, 난이도(Easy/Normal/Hard) 입력
- [ ] 퀘스트 목록 조회: 오늘의 퀘스트 최대 5개 표시
- [ ] 퀘스트 완료 처리: 완료 시 전투 보상 즉시 지급
- [ ] 하루가 지나면 퀘스트 목록 초기화

### M-02. 카드 전투 시스템
- [ ] 덱 구성: 초기 10장 (공격 5 + 방어 5)
- [ ] 턴 시스템: 플레이어 턴 → 몬스터 턴 반복
- [ ] 에너지 시스템: 턴당 에너지 3 충전, 카드 비용 차감
- [ ] 드로우 시스템: 턴 시작 시 5장 드로우, 덱 소진 시 버림 덱 셔플
- [ ] 방어도 시스템: 턴 종료 시 방어도 소멸
- [ ] 승패 판정: 몬스터 HP 0 → 승리, 플레이어 HP 0 → 패배

### M-03. 몬스터 AI
- [ ] 단일 몬스터 등장
- [ ] 의도(Intent) 표시: 공격 또는 방어
- [ ] 스테이지별 스탯 공식 적용 (HP = 20 + 스테이지×10)

### M-04. 캐릭터 상태
- [ ] HP 표시 및 관리 (최대 70)
- [ ] 스테이지 진행 상태 저장 (세션 내)

### M-05. 기본 카드 구현
- [ ] Strike (강타): 1에너지, 6 데미지
- [ ] Defend (방어): 1에너지, 방어도 5
- [ ] Bash (맹타): 2에너지, 8 데미지 + 취약 2턴
- [ ] Iron Wall (철벽): 2에너지, 방어도 10

## Should Have (권장 기능)

### S-01. 상태 이상
- [ ] 취약(Vulnerable): 받는 데미지 +50%
- [ ] 약화(Weak): 주는 데미지 -25%
- [ ] 상태 이상 아이콘 및 턴 수 UI 표시

### S-02. 보상 시스템
- [ ] 카드 선택 보상: 3장 중 1장 선택하여 덱에 추가
- [ ] 골드 시스템: 보상으로 골드 획득 (현재 버전에서는 표시만)
- [ ] 특수 카드 보상: Focus, Recover, Swift Cut

### S-03. 로그라이크 진행
- [ ] 스테이지 1~3 + 보스 구조
- [ ] Soft Reset: 패배 시 HP 절반, 스테이지 1로

### S-04. 데이터 영속성
- [ ] 앱 종료 후 재시작 시 퀘스트 목록 유지
- [ ] 캐릭터 HP 유지

### S-05. 몬스터 의도 다양화
- [ ] 공격 의도 / 방어 의도 외 버프 의도 추가

## Could Have (선택 기능)

### C-01. 유물(Relic) 시스템
- [ ] Hard 퀘스트 완료 보상으로 유물 획득
- [ ] 유물: 패시브 효과 부여 (예: 턴 시작 시 HP 1 회복)

### C-02. 덱 뷰어
- [ ] 현재 덱 구성 카드 목록 확인 화면

### C-03. 통계 화면
- [ ] 총 완료 퀘스트 수, 처치 몬스터 수, 연속 완료일 표시

### C-04. 애니메이션
- [ ] 카드 사용 시 슬래시 애니메이션
- [ ] 데미지 숫자 팝업 애니메이션

### C-05. 사운드 효과
- [ ] 카드 사용, 데미지, 승리/패배 효과음

---

## Won't Have (현재 버전 제외)

### W-01. 멀티플레이어
- 온라인 대전, 길드, 리더보드 등 — 네트워크 기능 전면 제외

### W-02. 캐릭터 선택
- 단일 캐릭터만 지원. 직업 시스템은 v2.0 이후 고려

### W-03. 인앱 결제 / 광고
- 완전 무료, 광고 없음. 상업적 수익 구조 배제

### W-04. 클라우드 동기화
- 로컬 저장만 지원. 계정 시스템 없음

### W-05. 스토어 출시 파이프라인
- CI/CD, 앱 서명, 스토어 등록 등 배포 인프라 제외
