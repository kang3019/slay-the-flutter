# 01-requirements.md — 요구사항 정의 (MoSCoW)

**버전**: 2.0 | **날짜**: 2026-05-19

---

## 핵심 사용자 시나리오

### 시나리오 1: 던전 진입과 기본 전투

> 플레이어가 게임을 처음 시작해 스테이지 1 몬스터와 첫 전투를 치른다.

1. 앱 실행 → 메인 화면에서 **"새 게임 시작"** 탭
2. 초기 덱(Strike×5, Defend×5) 10장이 자동 배정됨
3. 스테이지 1 몬스터 등장 (기본형 HP 24, 공격력 10 / 네임드 몬스터는 고정 HP)
4. 턴 시작: 에너지 3 충전, 5장 드로우
5. 플레이어가 Strike 카드(비용 1)를 드래그해 몬스터에 사용 → 6 데미지
6. Defend 카드(비용 1)를 사용해 방어도 5 획득
7. 턴 종료 → 몬스터 턴 (의도 아이콘으로 예고된 행동 실행)
8. 반복 끝에 몬스터 HP 0 → **승리** → 카드 보상 3장 중 1장 선택

**핵심 검증 포인트**: 에너지 차감, 드로우 사이클, 방어도 턴 종료 소멸, 승패 판정

---

### 시나리오 2: 보스전 패배와 메타 경험치 획득

> 스테이지 3까지 진행한 플레이어가 보스에게 패배하고, 메타 보상을 확인한다.

1. 스테이지 1→2→3을 클리어하며 카드 보상으로 덱을 강화
2. 보스 등장 (HP 96, 패턴 기반 행동: 장갑 강화 → 분쇄 강타 → 연속 타격 순환)
3. 덱 구성 실수 또는 운 부족으로 플레이어 HP 0 → **패배**
4. 패배 결과 화면: 클리어 스테이지, 처치 몬스터 수, 획득 XP 표시
5. XP가 레벨업 임계치를 넘지 않더라도 누적 저장됨
6. 메인 화면 복귀 → 레벨 바에 XP 반영

**핵심 검증 포인트**: 런 데이터 집계, XP 영속 저장, Soft Reset(HP 절반·스테이지 1) 동작

---

### 시나리오 3: 레벨업 및 새 카드 해금

> 누적 XP가 임계치를 초과해 레벨업하고, 잠겨 있던 카드가 보상 풀에 추가된다.

1. 이전 런에서 쌓인 XP가 임계치(예: 100 XP) 초과 → 레벨업 연출
2. 레벨업 보상 화면: 새로 해금된 카드 1~2장 표시 (예: 혈기(Blood Rush), 극한 각성(Limit Break))
3. 다음 런에서 카드 보상 선택지에 해금 카드가 포함됨
4. 플레이어가 혈기(Blood Rush, 에너지 X, X×6 데미지)를 보상으로 획득
5. 해금 카드로 강화된 덱으로 더 높은 스테이지 도전

**핵심 검증 포인트**: XP 누적 → 레벨업 트리거, 카드 해금 상태 영속 저장, 보상 풀 갱신

## MoSCoW 우선순위 설명

| 레벨 | 의미 |
|------|------|
| **Must Have** | 반드시 구현. 없으면 게임이 성립되지 않음 |
| **Should Have** | 강력히 권장. 없으면 로그라이크 핵심 경험이 저하됨 |
| **Could Have** | 있으면 좋음. 시간 여유 시 구현 |
| **Won't Have** | 현재 범위에서 제외. 미래 버전 고려 |

## Must Have (핵심 기능)

### M-01. 카드 전투 시스템

- [x] 덱 구성: 초기 10장 (Strike×5, Defend×5)
- [x] 턴 시스템: 플레이어 턴 → 몬스터 턴 반복
- [x] 에너지 시스템: 턴당 에너지 3 충전, 카드 비용 차감
- [x] 드로우 시스템: 턴 시작 5장 드로우, 덱 소진 시 버림 덱 셔플
- [x] 방어도 시스템: 턴 종료 시 방어도 0으로 초기화
- [x] 승패 판정: 몬스터 HP 0 → 승리 / 플레이어 HP 0 → 패배

### M-02. 기본 카드 구현

- [x] **Strike** (강타): 비용 1, 데미지 6
- [x] **Defend** (방어): 비용 1, 방어도 5
- [x] **Bash** (맹타): 비용 2, 데미지 8 + 취약 2턴
- [x] **Iron Wall** (철벽): 비용 2, 방어도 10

### M-03. 몬스터 AI

- [x] 단일 몬스터 등장 per 전투
- [x] 의도(Intent) 아이콘 표시: 공격 / 방어
- [x] 스테이지별 스탯 공식 (MonsterType.basic): HP = 16 + (stage × 8), 공격 = 8 + (stage × 2)

### M-04. 플레이어 상태 관리

- [x] HP 표시 및 관리 (최대 70)
- [x] 현재 스테이지 진행 상태 세션 내 보존

### M-05. 로그라이크 진행 구조

- [x] 스테이지 1 → 2 → 3 → 보스 순서 진행 (12층 맵, 절차적 생성)
- [x] 전투 승리 시 카드 보상 3장 중 1장 선택
- [ ] 패배 시 Soft Reset: HP 절반 · 스테이지 1 복귀 *(현재: 런 종료 후 새 런 시작)*

### M-06. XP / 레벨 시스템

- [x] 런 종료 시 XP 산정 및 누적 저장
- [x] 레벨업 시 카드 해금 상태 영속 저장
- [x] 해금 카드가 이후 런의 보상 풀에 포함

## Should Have (권장 기능)

### S-01. 상태 이상 (Status Effects)

- [x] **취약(Vulnerable)**: 받는 데미지 ×1.5
- [x] **약화(Weak)**: 주는 데미지 ×0.75 (내림)
- [x] **독(Poison)**: 턴 시작 시 스택만큼 HP 감소 (블록 무시)
- [x] 상태 이상 아이콘 및 잔여 턴 수 UI 표시

### S-02. 보스 고유 패턴

- [x] 보스는 일반 몬스터와 다른 공격 패턴 보유 (IronGolem: 방어→공격→공격 사이클, 장갑 강화 → 분쇄 강타 → 연속 타격)
- [x] 보스 전용 스탯 (HP 96, 패턴 기반 행동)

### S-03. 특수 카드

- [x] **Focus**: 비용 0, 다음 카드 효과 +50%
- [x] **Recover**: 비용 2, HP 8 회복
- [x] **Swift Cut**: 비용 1, 데미지 4 × 2회 (총 8)
- [x] 추가 구현: RageBurst, ToxicJab, Regroup, CrushingBlow, Fury, TripleSlash, QuickMend, SwiftGuard, ExploitWeakness, Sharpen, WeakSlash, BlockStrike, BloodRush, DevilsDeal, BattleCry, Indomitable, ComboStrike, Gamble, PoisonDart, LimitBreak, Impervious, DoubleTap, FiendFire (총 30종)

### S-04. 유물(Relic) 시스템

- [x] 전투 보상(엘리트 처치 시 자동 지급), 보스 처치 시 자동 지급, 유물 보관소 노드, 상점에서 유물 획득
- [x] 유물 패시브 효과 20종 이상 구현 (healOnTurnStart, firstAttackBonus, blockPerRemainingEnergy 등)

### S-05. 데이터 영속성

- [x] XP · 레벨 · 해금 카드 목록 로컬 저장 (앱 재시작 후 유지)
- [x] 진행 중인 런 상태 저장 (SaveSlot 3슬롯, JSON 직렬화)

## Won't Have (현재 버전 제외)

### W-01. 멀티플레이어

- 온라인 대전, 길드, 리더보드 등 네트워크 기능 전면 제외

### W-02. 캐릭터 선택 / 직업 시스템

- 단일 캐릭터만 지원. 직업 분기는 v2.0 이후 고려

### W-03. 인앱 결제 / 광고

- 완전 무료, 광고 없음. 상업적 수익 구조 배제

### W-04. 클라우드 동기화

- 로컬 저장만 지원. 계정 시스템 없음

### W-05. 스토어 출시 파이프라인

- CI/CD, 앱 서명, 스토어 등록 등 배포 인프라 제외
