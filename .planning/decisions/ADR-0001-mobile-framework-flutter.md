# ADR-0001: 모바일 프레임워크로 Flutter 선택

## 상태

**수용됨** — 2026-05-19

---

## 배경

Slay the Flutter는 Android 및 iOS를 동시에 지원해야 하는 모바일 덱빌딩 카드 게임이다. 게임의 핵심 경험은 카드 드래그, 타격 이펙트, 상태 이상 애니메이션 등 풍부한 인터랙션에 의존하므로, 프레임워크의 렌더링 성능과 커스텀 애니메이션 지원 수준이 기술 선택의 핵심 기준이 되었다.

플랫폼별 네이티브 개발(Swift/Kotlin), React Native, Flutter 세 가지 옵션을 검토하였다.

---

## 결정 사항

**프론트엔드 프레임워크로 Flutter(Dart)를 채택한다.**

모든 UI는 Flutter의 자체 렌더링 엔진(Impeller/Skia) 위에서 구현하며, 플랫폼별 분기는 최소화한다.

---

## 대안 및 선택 이유

### 검토된 대안

| 옵션 | 설명 | 탈락 이유 |
|------|------|-----------|
| Swift + Kotlin (네이티브 이중 개발) | 각 플랫폼 최고 성능 | 동일 로직을 두 번 구현해야 하므로 유지비용이 2배가 되며, 소규모 팀에 비현실적 |
| React Native | JS 기반 크로스플랫폼 | 브리지 레이어를 통한 네이티브 컴포넌트 위임 구조상, 60fps 이상의 카드 애니메이션을 일관되게 보장하기 어렵고 커스텀 렌더링 파이프라인 구성이 복잡함 |

### Flutter를 선택한 이유

1. **단일 코드베이스로 iOS/Android 동시 대응** — 두 플랫폼 간 동작 차이가 거의 없어 QA 비용이 절감된다.
2. **자체 렌더링 엔진** — 네이티브 위젯을 브리지하지 않고 GPU 직접 렌더링(Impeller)을 수행하므로, 카드 뒤집기·타격 이펙트·파티클 등 복잡한 애니메이션을 안정적으로 60fps로 구현할 수 있다.
3. **`AnimationController` / `CustomPainter` 생태계** — 카드 게임에 필요한 커스텀 드래그, 스택 레이아웃, 트윈 애니메이션을 프레임워크 수준에서 직접 제공한다.
4. **Dart의 강타입 시스템** — 게임 도메인(카드 효과, 상태 이상 열거형 등)의 타입 안정성을 컴파일 타임에 검증할 수 있어 런타임 오류를 사전에 차단한다.

---

## 결과

### 장점

- Android/iOS 코드를 단일 저장소에서 관리하여 개발 속도가 향상된다.
- 렌더링 파이프라인을 직접 제어할 수 있어 카드 게임 특유의 시각적 피드백(타격감, 이펙트)을 자유롭게 구현할 수 있다.
- `flutter test` 기반의 풍부한 위젯 테스트 인프라를 활용할 수 있다.

### 단점

- Dart는 JavaScript/Kotlin 대비 생태계 규모가 작아, 특수 네이티브 기능(예: 고급 Haptic 제어)은 플러그인 품질에 의존해야 한다.
- 앱 번들 크기가 네이티브 대비 소폭 크다.
- 팀원이 Dart/Flutter에 익숙하지 않을 경우 초기 학습 비용이 발생한다.
