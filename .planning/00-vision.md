# 00-vision.md — 프로젝트 비전

## 한 줄 요약

> "런마다 새로운 덱을 구성하고, 레벨업으로 더 많은 카드와 유물을 해금하며 던전을 공략하는 미니멀 덱빌딩 로그라이크 — 덱빌딩 로그라이크 장르에서 영감을 받아 모바일에 맞게 새롭게 해석한 게임"

---

## 왜 만드는가 (Why)

덱빌딩 로그라이크 장르는 "이번 런은 어떤 덱을 만들어볼까"라는 질문 하나로 강력한 반복 플레이 동기를 만든다. **슬레이 더 플러터**는 이 핵심 재미를 모바일에서 미니멀하게 구현하는 것을 목표로 한다:

1. **런 내 성장** — 매 런마다 카드를 고르고 덱을 구성하며 스테이지를 돌파한다.
2. **메타 성장** — 런에서 쌓은 경험치로 레벨업하면 새로운 카드와 유물이 영구 해금된다.
3. **빌드 다양성** — 해금된 카드 풀이 넓어질수록 시도할 수 있는 덱 아키타입이 늘어난다.

---

## 무엇을 만드는가 (What)

- Flutter 기반 크로스플랫폼 모바일 게임 (Android / iOS 우선)
- 턴제 카드 전투 + 덱빌딩 + 로그라이크 진행 구조
- 미니멀한 디자인: 복잡한 메뉴 없이 런 시작 → 전투 → 보상 선택 → 클리어의 명확한 루프
- 로컬 데이터 저장 (경험치, 레벨, 해금 목록 영구 보존)

---

## 핵심 게임 루프

```
[런 시작]
  └─ 초기 덱(기본 카드)으로 시작
       │
[스테이지 전투]
  └─ 카드를 사용해 몬스터 처치
       │
[보상 선택]
  └─ 카드 추가 / 유물 획득 / HP 회복 중 선택
       │
[반복] → 보스 처치 → 런 클리어
                   → 패배 → 런 종료
       │
[경험치 획득]
  └─ 런 결과(클리어 여부, 처치 몬스터 수 등)에 따라 XP 지급
       │
[레벨업]
  └─ 새 카드 또는 유물 영구 해금 → 다음 런에서 등장 가능
```

---

## 메타 진행 (영구 성장)

- **경험치(XP)**: 런 클리어, 스테이지 클리어, 몬스터 처치 등으로 획득
- **레벨업**: XP 누적 시 레벨 상승, 레벨마다 해금 항목 1~2개 선택
- **해금 대상**: 카드(공격/방어/특수) 또는 유물(패시브 효과 아이템)
- **초기 상태**: 기본 카드(강타, 방어)만 존재 → 레벨업으로 점차 풀 확장

---

## 누구를 위한 것인가 (Who)

- **주요 타깃**: 덱빌딩 로그라이크 장르를 좋아하지만 모바일에서 가볍게 즐길 수 있는 게임을 찾는 사람
- **2차 타깃**: 로그라이크 장르를 처음 접하는 사람 — 미니멀한 규칙으로 진입 장벽을 낮춤

---

## 성공 기준 (Definition of Done)

| 기준 | 목표 |
|------|------|
| 핵심 전투 루프 | 카드 사용 → 데미지 → 몬스터 처치 → 보상 선택 동작 |
| 런 진행 구조 | 스테이지 1→2→3→보스 클리어 후 런 종료 |
| 메타 진행 | XP 획득 → 레벨업 → 카드/유물 해금 동작 |
| Layered Architecture | 레이어 간 의존성 역전 원칙 준수 |
| 테스트 커버리지 | 게임 로직 핵심 메서드 80% 이상 |
| 코드 품질 | `flutter analyze` 경고 0건 |

## 개발 단계 로드맵

```
Phase 1 — 기반 (완료)
  ├─ 프로젝트 스캐폴딩 및 문서화
  └─ Layered Architecture 폴더 구조 세팅

Phase 2 — 게임 코어
  ├─ 카드/캐릭터/몬스터 Model 구현
  ├─ 전투 로직 ViewModel 구현
  ├─ XP 및 레벨업 시스템 구현
  └─ 단위 테스트 작성

Phase 3 — UI 구현
  ├─ 전투 화면 (Battle View)
  ├─ 보상 선택 화면 (Reward View)
  ├─ 메타 진행 화면 (Level / Unlock View)
  └─ 런 시작 화면 (Run Start View)

Phase 4 — 완성
  ├─ 데이터 영속성 (경험치, 레벨, 해금 목록 저장)
  ├─ 카드/유물 풀 확장
  ├─ 애니메이션 및 피드백
  └─ 통합 테스트
```
