# 006 — "테스트 먼저"를 규칙 파일에 적어두면 매번 반복하지 않아도 된다

| 항목 | 내용 |

## 패턴

초기에는 새 게임 로직을 요청할 때마다 "테스트를 먼저 작성하고 구현해줘"를 매번 프롬프트에 적었다. 이를 `AGENTS.md` §4에 다음과 같이 **프로젝트 규칙으로 명문화**한 뒤부터는, 단순히 "독 상태이상 추가해줘"처럼 기능만 요청해도 AI가 먼저 `test/domain/...` 또는 `test/application/...`에 테스트를 작성하고, 실패를 거친 뒤 구현 코드를 작성하는 순서를 스스로 따랐다.

```markdown
## 4. 테스트 주도 개발 (TDD)
- 새로운 게임 로직 추가 시, 테스트를 먼저 작성한 후 구현 코드를 작성한다.
- test/ 디렉토리 구조는 lib/의 Domain·Application 계층을 미러링한다.
```

## 왜 효과가 있었는가

CLAUDE.md/AGENTS.md는 **모든 세션 시작 시 컨텍스트로 로드**된다. 즉, 한 번 규칙 파일에 적어두면 이후의 모든 대화에서 반복 설명할 필요가 없다 — "매번 같은 지시를 타이핑하고 있다"는 것 자체가 "규칙 파일에 옮겨야 한다"는 신호다.

반대로, 규칙 파일에 없는 새로운 습관(예: "비주얼 변경은 스크린샷과 함께 요청" — [003](003-visual-bug-iteration.md) 참고)은 매번 다시 설명해야 했다.

## 배운 것

> 같은 지시를 두 번째 입력하고 있다면, 그 지시는 프롬프트가 아니라 **CLAUDE.md/AGENTS.md에 적어야 할 규칙**이다.

이 프로젝트에서는 TDD 외에도 "Notifier/AsyncNotifier만 사용", "매직 넘버 상수화", "`flutter analyze` 0 warnings" 등을 모두 AGENTS.md에 명문화해, 매 기능 요청을 짧게 유지하면서도 일관된 결과를 얻고 있다.
