⏱ 1:44–2:06
ADR Summary · Q&A 대비
아키텍처 의사결정 기록 (ADR)
ADR-0001
플랫폼 — Flutter
React Native·Android/iOS 단독·KMP 대비, 단일 코드로 양 플랫폼 + 커스텀 카드 UI 자유도 확보
✅ 결과: 4-Layer 구조를 언어 차원에서 강제, flutter test·analyze 파이프라인 즉시 사용
ADR-0002
아키텍처 — 4-Layer + Riverpod
MVC·MVP·MVVM·Clean Architecture 대비, Use Case를 Application에 통합해 1인 개발 규모에 최적화
✅ 결과: domain 테스트 커버리지 80%+ 달성, 화면 변경이 게임 규칙에 영향 없음
ADR-0003
상태관리 — Riverpod
Provider·Bloc·GetX 대비, BuildContext 없이 ProviderContainer로 테스트 격리 가능
✅ 결과: 위젯 없이 Notifier 단위 테스트, AsyncValue로 로딩·에러 상태를 컴파일러가 강제 검증
ADR-0004
영속성 — SharedPreferences
Firestore·SQLite 대비, 키-값 규모 데이터에 적합한 로컬-퍼스트·완전 오프라인
✅ 결과: 서버·계정 없이 완전 오프라인 동작, 기기 변경 시 데이터 소실은 트레이드오프로 감수
ADR-0005
맵 렌더링 — CustomPaint
Row/Column+Stack·graphview 대비, computeNodePositions를 SSOT로 공유해 그리기=탭 일치
✅ 결과: 그리기 좌표 = 탭 인식 좌표 항상 일치, 좌표 계산을 순수 함수로 단위 테스트 가능

Build & Deploy
빌드와 배포 과정
빌드 vs 배포
빌드 = 실행 가능한 산출물(APK/IPA/정적 파일) 생성  |  배포 = 그 산출물을 사용자·스토어·서버에 전달

🤖 Android

flutter build apk --release
flutter build appbundle --release
→ app-release.apk / .aab
⚠ 현재 debug 서명 사용 (Out of Scope)

🍎 iOS

flutter build ipa --release
→ build/ios/ipa/*.ipa
Xcode에서 Signing & Capabilities 설정 필요

🌐 Web

flutter build web --release
→ build/web/ 정적 파일
GitHub Actions가 docs/ → GitHub Pages 자동 배포

Trial & Error

🎨 비주얼 버그 왕복 수정

문제 피격 시 빨간 그라데이션을 "안 보인다→너무 크다→다시 좁게" 3연속 수정.
원인 "느낌" 위주 피드백은 AI가 목표 수치를 가늠할 수 없어 무작위 왕복으로 이어짐.
교훈 목표 비율·수치 또는 스크린샷으로 전달.

🔌 보상 연결 누락

문제 신규 카드 18종을 보상 풀에 추가했지만, 실제 전투에 보상 카드·회복 HP가 반영되지 않음.
원인 단위 테스트는 통과했지만 BattleEngine.start() 진입점 파라미터 연결을 놓침.
교훈 런을 처음부터 끝까지 직접 플레이해 검증.

🗺️ 절차적 맵 생성 검증

문제 절차적 DAG 맵 생성이 5가지 제약을 만족하는지 AI 설명만으로는 신뢰 불가.
원인 무작위성 코드의 "만족합니다"는 의도일 뿐, 모든 실행의 보장이 아님.
교훈 100회 반복 생성 + 전수 검사 테스트로 엣지 케이스 발견·수정.
