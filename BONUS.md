# BONUS.md — 가산점 트래킹 (최대 +6점)

**버전**: 1.0 | **날짜**: 2026-05-19

10주차 평가 루브릭 기준 가산점 항목을 트래킹합니다.  
각 항목을 달성하면 체크박스를 체크하고, 근거 링크(커밋 해시·파일 경로)를 함께 기록합니다.

---

## A. AI Agent / 워크플로우 적극 활용 (+1점)

> 단순 코드 생성이 아닌, AI를 **설계·검토·자동화 워크플로우**에 통합했음을 증명

- [ ] Claude Code(또는 동급 AI)로 요구사항·위험 문서 초안 생성 후 직접 검토·수정
  - 근거:
- [ ] AI가 생성한 코드에 대해 TDD(테스트 선작성)를 적용한 PR이 1개 이상 존재
  - 근거:
- [ ] AI 워크플로우(프롬프트 → 리뷰 → 병합) 흐름이 ADR 또는 커밋 메시지에 기록됨
  - 근거:

**달성 여부**: ☐ 미달성 / ☑ 달성 (달성 시 체크)

---

## B. 본인만의 기법 구성 (+2점)

> 강의 내용을 넘어서 **직접 설계하거나 응용한 독창적 기법**이 코드·문서에 녹아 있음

- [ ] MVVM + Riverpod 패턴을 로그라이크 턴제 전투에 맞게 직접 설계 (BattleViewModel 구조)
  - 근거:
- [ ] 카드 효과를 `CardEffect` 인터페이스로 추상화해 확장 가능한 구조로 구현
  - 근거:
- [ ] 기존 라이브러리를 사용하지 않고 덱 셔플·드로우 사이클을 순수 Dart로 구현하고 단위 테스트 작성
  - 근거:
- [ ] 본인이 직접 도출한 게임 공식(데미지 계산, 상태 이상 배율 등)을 상수화하고 ADR에 근거 기록
  - 근거:

**달성 여부**: ☐ 미달성 / ☑ 달성

---

## C. 본인 암묵지(LLM Wiki) 운영 (+3점)

> AI와의 협업 과정에서 얻은 **개인 지식·실패 경험·설계 결정**을 지속적으로 문서화

- [ ] `docs/llm-wiki/` 폴더를 생성하고 주 1회 이상 항목 추가
  - 근거:
- [ ] LLM이 틀리거나 한계를 보인 사례를 1건 이상 기록 (예: 잘못된 Riverpod API 제안)
  - 근거:
- [ ] 직접 수정·보완한 AI 코드에 대해 "왜 바꿨는지"를 Wiki 또는 ADR에 기록
  - 근거:
- [ ] ADR(Architecture Decision Records) 3건 이상 작성 (`.planning/adr/` 참고)
  - 근거:
- [ ] Wiki 항목이 다른 사람이 읽어도 이해할 수 있도록 작성됨 (제3자 가독성)
  - 근거:

**달성 여부**: ☐ 미달성 / ☑ 달성
